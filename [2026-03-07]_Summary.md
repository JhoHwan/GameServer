# 오늘 작업 요약 (2026-03-08)

## 1. NavMesh 기반 비동기 길 찾기 연산 적용 및 최적화
- `FieldInstance` 내에 `PlayerRequestMove` 로직을 구현하여 클라이언트 이동 요청(Click) 시 Detour의 `dtNavMeshQuery`를 이용해 목적지까지의 경로(Waypoints)를 찾아내는 연산을 성공적으로 적용함.
- `JobQueue` 시스템 내에서 단일 틱으로 길 찾기를 수행했음에도 첫 연산 1.2ms, 이후 캐시 히트로 0.1ms대의 뛰어난 성능을 기록하여 별도의 Compute 스레드 분리 없이도 현재 필드 큐에서 안전하고 빠르게 처리됨을 증명함.
- 연산된 경로 배열을 언리얼 클라이언트가 보간 이동할 수 있도록 `SC_MOVE_PATH` 패킷에 담아 응답 및 주변 유저에게 브로드캐스트하도록 구현.

## 2. 언리얼 엔진 호환성을 위한 클라이언트 패킷 제너레이터 리팩토링
- 프로토버프 패킷 자동 생성 스크립트(`PacketGen.bat`)를 언리얼 엔진의 C++ 모듈 시스템(Public/Private 폴더 분리) 규칙에 완벽히 맞도록 대대적으로 수정.
- `protoc` 실행 옵션에 `dllexport_decl=GAMENET_API` 매크로를 삽입하여, 언리얼 프로젝트 빌드 시 생성된 패킷 클래스(`.pb.cc/h`)들이 외부 모듈(DLL)에서도 안전하게 참조(Export)될 수 있도록 구조를 정립함.
- 생성 후 헤더(`.h`) 파일은 `Public` 폴더로, 소스(`.cc`) 파일은 `Private` 폴더로 자동 분리/이동되도록 파이프라인 자동화 완료.

## 3. 통합 LogManager 시스템 도입 및 매크로 확장
- C++20 `std::format`을 활용하여 타입에 구애받지 않고 가독성 높은 `LOG_DEBUG`, `LOG_INFO`, `LOG_WARN`, `LOG_ERROR` 매크로를 구축함.
- `ConcurrentQueue`와 전용 스레드(`std::jthread`)를 활용한 락프리 기반의 비동기 파일/콘솔 로깅 시스템을 구현하여, 로깅으로 인한 게임 로직의 병목과 멈춤(Block) 현상을 제거함.
- Windows와 Linux의 `Session::Disconnect` 등 주요 네트워크 이벤트의 하드코딩된 출력을 모두 새 통합 로그 시스템으로 전환.

## 4. FieldManager 지연 로딩(Lazy Loading) 및 서버 초기화 안정성 개선
- 서버 구동 시 `main.cpp`에서 `LogManager`와 `FieldManager`의 초기화 실패 여부를 검증하고 오류 발생 시 즉각 종료하도록 Error Handling 강화.
- `FieldManager`를 리팩토링하여 서버 시작 시 네비게이션 맵(`Test.bin`)만 캐싱하고, 실제 `FieldInstance`는 유저가 입장(또는 접근)할 때 비로소 생성(Create)되는 동적 지연 로딩(Lazy Loading) 구조로 개편하여 메모리와 자원을 절약함.
- 게임 접속 및 이동 로직을 `GameManager` 싱글톤으로 위임하여 패킷 핸들러와의 책임을 깔끔하게 분리함.

## 5. JobQueue 캡슐화를 위한 AsyncActor 도입
- 객체가 상속받아 손쉽게 자신만의 락프리 큐를 가질 수 있도록, 내부에 `JobQueueRef`를 할당받아 관리하고 `DoAsync`를 제공하는 `AsyncActor` 래퍼(Wrapper) 패턴을 `ServerCore`에 도입함.

```cpp
// Server/GameServer/Contents/Field.cpp : 락프리 기반 필드 내부 길 찾기 연산
void FieldInstance::PlayerRequestMove(weak_ptr<PlayerCharacter> player, const Protocol::Vector3& dest)
{
    // FieldInstance의 JobQueue 내에서 실행 (단일 스레드 보장, Lock 불필요)
    DoAsync([this, player = std::move(player), dest]()
    {
        auto playerRef = player.lock();
        if(!playerRef) return;

        dtReal startPos[3] {playerRef->Transform()->GetPos().x, playerRef->Transform()->GetPos().z, playerRef->Transform()->GetPos().y};
        dtReal endPos[3] {dest.x(), dest.z(), dest.y()};

        dtQueryResult result;
        FindPath(startPos, endPos, result); // Detour NavMesh 연산 수행

        Protocol::SC_MOVE_PATH pkt;
        pkt.set_object_id(playerRef->GetId());

        // 찾아낸 경로 노드(Waypoints)들을 패킷에 적재
        for (int i = 0; i < result.size(); i++)
        {
            auto* pos = result.getPos(i);
            auto* waypoint = pkt.add_waypoints();
            waypoint->set_x(pos[0]);
            waypoint->set_y(pos[2]); // Y, Z축 언리얼 스위즐링 복원
            waypoint->set_z(pos[1]);
        }
        
        // 주변 유저에게 이동 경로 브로드캐스트
        BroadCast(ServerPacketHandler::MakeSendBuffer(pkt));
    });
}
```
