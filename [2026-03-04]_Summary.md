# 오늘 작업 요약 (2026-03-04)

## 1. 스레드 로컬 스토리지(TLS) 기반 락-프리 타이머 최적화
- `JobTimer` 객체를 전역(Global)에서 스레드 로컬 스토리지(TLS, `thread_local LJobTimer`)로 변경하여 스레드 간의 락(Lock) 경합을 완전히 제거함.
- `JobTimer`의 큐(`priority_queue`) 접근 시 락이 필요 없어지면서, 수만 개의 타이머 이벤트 예약(Reserve) 및 분배(Distribute) 시 발생할 수 있는 병목을 원천적으로 차단하여 최상의 동시성 성능을 확보함.

## 2. JobQueue 스케줄링 및 협력적 멀티태스킹 최적화
- `JobQueue`의 작업 처리 함수(`Execute`)가 특정 시간 기반이 아닌, 할당된 처리 개수(`executeCount`)를 기준으로 작동하도록 변경.
- 최대 실행 횟수를 초과할 경우 잔여 작업을 가진 `JobQueue` 자신을 글로벌 큐(`GGlobalJobQueue`)에 반납하고 스레드를 양보(Yield)함으로써, 특정 무거운 작업 큐로 인한 다른 큐의 기아 현상(Starvation)을 방지함.

## 3. 플랫폼별 Listener 클래스 물리적 분리 완료
- `Network/Listener.cpp` 파일 내에 `#ifdef _WIN32`로 섞여 있던 코드를 `Platform/Windows/Listener.cpp`와 `Platform/Linux/Listener.cpp`로 완전히 분할.
- **Windows (IOCP):** Proactor 패턴에 맞춘 `AcceptEx` 다중 투척 로직 전담.
- **Linux (Epoll):** Reactor 패턴에 맞춘 이벤트 통지 시점에서의 `while(true) accept4` 루프 전담.

## 4. 플랫폼별 Session 클래스 물리적 분리 및 전송 버퍼 구조 개선
- `Listener`와 마찬가지로 `Session.cpp`를 `Platform/Windows`와 `Platform/Linux`로 물리적으로 분리하여 플랫폼별 코어 로직의 가독성과 유지보수성을 극대화함.
- **리눅스 FlushSend 버그 수정 및 Lock-Free 적용:** 기존 `std::queue`에서 `front()` 요소만 참조하고 `pop()`을 하지 않아 발생할 수 있는 무한루프 취약점을 해결. 
- 큐 접근 최적화를 위해 ConcurrentQueue의 `try_dequeue_bulk`를 활용해 데이터를 한 번에 가져오고, 부분 전송(Partial Send) 시 남은 데이터를 임시 보관하기 위해 `_pendingSendQueue`(`std::deque`) 구조를 새롭게 도입하여 Lock 경합을 완전히 제거함.

void JobQueue::Execute(int32 executeCount)
{
    int currentJobCount = 0;
    while (true)
    {
        JobRef jobs[32];
        const auto jobCount = static_cast<int>(_jobs.try_dequeue_bulk(jobs, 32));
        // ... (작업 실행) ...
        currentJobCount += jobCount;
        if (currentJobCount >= executeCount) // 할당량 초과 시
        {
            GGlobalJobQueue.enqueue(shared_from_this()); // 글로벌 큐에 반납하고 스레드 양보
            return;
        }
    }
}

// Server/ServerCore/Platform/Linux/Session.cpp: 벌크 디큐 및 deque를 활용한 Partial Send 대응
void Session::FlushSend()
{
    while(true)
    {
        constexpr int MAX_IOV = 64;
        // Lock 없이 한 번에 가져오기
        _sendQueue.try_dequeue_bulk(back_inserter(_pendingSendQueue), MAX_IOV);

        if (_pendingSendQueue.empty()) { /* Double-check 로직 */ break; }

        // _pendingSendQueue (deque)에서 데이터 추출 후 writev 실행
        // ...
        
        auto bytesToRemove = static_cast<int32>(sentBytes);
        while (bytesToRemove > 0 && _pendingSendQueue.empty() == false)
        {
            const auto& buffer = _pendingSendQueue.front();
            int32 dataInBuf = buffer->WriteSize() - _sendOffset;
            if(bytesToRemove >= dataInBuf)
            {
                bytesToRemove -= dataInBuf;
                _pendingSendQueue.pop_front(); // 보낸 만큼 앞에서부터 제거
                _sendOffset = 0;
            }
            else
            {
                _sendOffset += bytesToRemove; // 부분 전송 시 오프셋만 증가
                bytesToRemove = 0;
            }
        }
    }
}
