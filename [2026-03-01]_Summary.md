# 오늘 작업 요약 (2026-03-01)

## 1. ServerCore 아키텍처 문서 고도화 및 지침 수립
ServerCore의 비동기 I/O 모델, Pimpl 패턴의 목적, 메모리 관리 지침(Pending I/O 중 해제 금지 등)을 포함하여 `Architecture.md` 파일을 최신화함. 크로스 플랫폼 개발 시 발생할 수 있는 Incomplete Type 문제에 대한 가이드라인을 명시함.

## 2. WSL 빌드 오류 해결 및 PCH 시스템 최적화
WSL(Linux) 환경에서 발생하던 대규모 재정의(Redefinition) 에러의 원인이 헤더 파일의 UTF-8 BOM 마커임을 식별하고 이를 일괄 제거함. 또한 CMake의 자동 PCH 주입과 충돌하던 각 .cpp 파일의 수동 `#include "pch.h"`를 삭제하여 컴파일 안정성을 확보함.

## 3. MSVC 빌드 환경 및 인코딩 이슈 해결
CLion/Ninja 환경에서 MSVC 빌드 메시지가 깨지는 문제를 해결하기 위해 `VSLANG=1033` 설정을 추가함. 또한 BOM 없는 UTF-8 파일에서 발생하는 C4819 경고를 끄고 소스 코드를 UTF-8로 강제 해석하도록 `/wd4819 /utf-8` 옵션을 반영함.

## 4. 리눅스 Epoll 아키텍처 설계 및 수명 관리 로직 구현
리눅스 환경의 핵심인 `EpollCore`를 구현함. 멀티스레드 Dispatch 과정에서 발생할 수 있는 댕글링 포인터 문제를 방지하기 위해 `NetObject`가 Epoll 등록 시 자기 자신을 참조하는 `_epollRef` (shared_from_this) 로직을 도입함.

## 5. 성능 최적화를 위한 pImpl 리팩토링 철회 및 구조 복구
Session과 Listener에 적용했던 pImpl 패턴이 고성능 네트워크 엔진에서 포인터 체이싱 및 캐시 미스를 유발할 수 있음을 고려하여 이를 철회함. 플랫폼 종속 로직을 다시 본체 클래스로 통합하여 런타임 성능을 최우선으로 하는 기존 구조로 복구 완료함.

```cpp
// Server/ServerCore/Platform/Linux/EpollCore.cpp: 안전한 객체 등록 로직
bool EpollCore::Register(NetObjectRef netObject)
{
    const SOCKET socket = netObject->GetHandle();
    epoll_event event{};
    event.events = EPOLLIN | EPOLLET;
    event.data.ptr = netObject.get(); // 로우 포인터 전달

    // epoll_ctl 성공 시에만 스스로의 참조를 유지하여 수명 보장 (댕글링 방지)
    if (epoll_ctl(_epoll_fd, EPOLL_CTL_ADD, socket, &event) == -1)
        return false;

    netObject->_epollRef = netObject; 
    return true;
}
```
